# 实战子目录：把设计模式用在真实开发

这个子目录聚焦“模式落地”，提供可以直接参考的业务场景示例和落地清单，帮你把仓库里的 23 个设计模式应用到真实需求中。

## 目录结构

- 每个场景单独一个文件夹，包含 `README.md`（讲业务背景、用到的模式、扩展思路）与可运行的示例代码。
- 当前示例：`order_checkout/`，演示结算流程如何组合策略模式、工厂方法、观察者、命令等模式。

## 适合这样用

- 想快速把模式嵌入业务代码：照搬示例中的模式组合方式和接口分层。
- 需要给团队培训或做模式评审：用场景 README 里的决策理由作为讲解材料。
- 想验证某个模式是否合适：先在这里做一个最小可行用例再迁移到主工程。

## 模式落地速查

- 可插拔规则/算法：策略模式（如动态切换定价、路由、风控规则）。
- 统一实例创建且便于替换实现：工厂方法/抽象工厂（如支付、存储、消息通道选择）。
- 事件驱动与解耦通知：观察者（如审计、埋点、告警监听）。
- 串行/可回滚的工作流：命令模式、责任链（如订单履约、审批流）。
- 复杂对象或配置构建：生成器 + 组合（如场景/管线拼装）。
- 资源复用与性能：享元、对象池（如连接池、缓冲区池）。

## 如何添加新场景

1) 在本目录下创建子文件夹（如 `inventory_sync/`）  
2) 写一份场景 README，说明业务目标、使用的模式、扩展点  
3) 提供可运行的示例（建议可用 `g++ -std=c++14 -pthread -I./include ...` 编译）  
4) 在下方“场景列表”补充一行介绍

## 场景列表

- `order_checkout/`：订单结算与履约流水线，涵盖定价策略选择、支付渠道工厂、事件通知、命令式工作流。
